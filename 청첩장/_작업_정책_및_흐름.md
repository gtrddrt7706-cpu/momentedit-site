## 4. 부모 이름 표시 정책 (실무자 토글 항목)

청첩장에 부모 이름이 들어가는 위치는 **2곳**:
- **위쪽** — 인사글 끝부분의 자녀 소개 부분
- **아래쪽** — 마음 전하실 곳 (계좌 안내)

### 토글 항목 (2개)
| 시트 컬럼 | 값 | 결과 |
|---|---|---|
| `greetingShowParents` | YES | 인사글에 부모 이름 표시 |
| | NO | 인사글에 본인 이름만 표시 |
| `envelopeShowParents` | YES | 계좌 안내에 부모 계좌까지 표시 |
| | NO | 계좌 안내에 본인 계좌만 표시 |

→ 2 × 2 = **4가지 경우**:
| # | 위쪽 | 아래쪽 | 실무 빈도 |
|---|---|---|---|
| 1 | ✅ 부모 이름 | ✅ 부모 계좌 | 가장 흔함 (기본) |
| 2 | ✅ 부모 이름 | ❌ 본인 계좌만 | 흔함 (부모 자금 분리) |
| 3 | ❌ 본인만 | ✅ 부모 계좌 | 드묾 (모던 인사글 + 부모 계좌) |
| 4 | ❌ 본인만 | ❌ 본인만 | 흔함 (미니멀 톤) |

### 04, 08 특이 사항 (2026.05.25 정정)
- **8종 모두 인사글(초대의 글) 영역이 있다.** 과거 "04·08은 인사글 영역 없음"은 옛 상태로, 현행 배포 템플릿(`i/cover-04.html`·`i/cover-08.html`)에는 `OPTIONAL:invitationText` 영역이 존재함(04는 "초대의 글", 08도 "초대의 글" 섹션).
- 02 Editorial은 추가로 표지 대표 문구(`pullQuote`), 08 Noir는 추가로 "두 사람" 소개 카드(`groomBio`/`brideBio`)를 가진다 — 인사글과 별개 영역.
- `invitationText`는 비우면 디자인별 기본 인사글, 직접 입력 시 치환 (8종 공통).

## 11. 자동화 시스템 현황 (2026-05-25 기준) — 구글폼 → 청첩장

### 전체 흐름
구글폼 1개 작성 → (Apps Script `onCoupleFormSubmit` 트리거) **Couples 시트 자동 기록** → 디지털/가족 청첩장 **URL 자동 조립** → 제출 즉시 부부 이메일로 **URL 자동 발송**. 고객 수정은 **폼 재작성**(같은 성함·날짜면 같은 행 갱신). 사람이 직접 할 일은 예식 영상(Vimeo)뿐.

### Apps Script 2개 (둘 다 Couples 스프레드시트에 바인딩, contact@ 별칭으로 메일 발송)
1. **편지 시스템** — `청첩장/빌드.txt`가 참조본(Code.gs v3.3). `doGet ?action=getCouple`(부부정보 JSON, hydrate가 호출) + `doPost`(하객 편지 금지어검사·메일). 시트 **HEADER_ROW=3**(영문 헤더), 데이터 **row 4+**.
2. **폼 자동화** — `automation/form-to-couple.gs`. `createCoupleForm()`(폼 생성·트리거 등록) + `onCoupleFormSubmit`(시트 기록·URL·메일). **본 자동화의 핵심 파일.**

### form-to-couple.gs 핵심 동작
- 예식ID 자동생성(영문 이니셜 + MMDD, 충돌 시 `-2`). 캐시 무효화로 재제출 즉시 반영.
- **분기**: ① 인사말 혼주 성함 게이트 ② 계좌 부모 표시 게이트(독립) ③ 디지털 게이트(만들기 / 안 함 / **입장 QR만**) ④ 02·08 전용 질문은 **디지털 디자인 선택 시에만** 노출.
  - ⚠️ 가족만 02·08이면 기본 문구 사용 — 구글폼은 "온라인 OR 가족" 동시조건 표시 불가.
- 디자인 선택지 라벨 **"01번"**(제출값에서 숫자만 추출, `pad2`).
- **자동메일**: 부부에게 청첩장 URL + 폼 재작성 안내. `STUDIO_EMAIL`(contact@) 발송.
- **폼 안내 이미지**: `CFG.IMG_*`에 URL 또는 드라이브 파일ID. 현재 **raw.githubusercontent**(공개 저장소) 사용 — `momentedit.kr`은 서버측 fetch를 **403 차단**하므로 그 도메인 URL은 폼에서 못 가져옴.

### hydrate.js (청첩장 렌더 · 16개 공통, shared/)
- Couples 데이터 → cover/family HTML의 `{{변수}}`·`OPTIONAL` 치환.
- **기본값(비우면 자동)**: 인사말(디자인별 기본) · 02 `pullQuote` 기본("서로의 가장 진실한 / 순간을 기록하기로 합니다.") · 08 한마디 기본(신랑 "특별하지 않은 하루도, 함께라 충분합니다." / 신부 "사랑한다는 말보다, 더 오래 곁에 있겠습니다.").
- 직접작성 인사말 `*별표*` → 강조(`.em`). **전 16개 디자인 `.em` 정의 확인됨.**
- 프리뷰 더미(?e 없을 때): **이서준·정하윤** / 이재환·최미경 · 정영석·박윤희 / 2026-10-24.

### 배포 / 빌드
- 사이트: **Vercel 자동배포**(main 머지 시). 변경은 작업브랜치 → PR → **main 머지**.
- 갤러리(`invitation-gallery.html`): `청첩장/마스터/build_gallery.py --apply`로 재빌드(디자인을 base64로 임베드 + `.bak` 백업). **DATA의 더미·기본문구는 hydrate와 수동 동기화 필요**(자동 아님).
