# 계약서 요청 폼이 마이페이지에서 사라진 문제 — 진단과 해결

## 질문

> 지금 계약서 요청 단계인데 (마이페이지에서) **요청 항목이 안 나온다.**
> 전에는 나왔었는데 지금 다시 안 나온다. 왜 이러지?

### 관찰된 증상
- 진행바는 **상담완료(= 계약서 요청)** 단계, "다음 · 계약서 서명"으로 정상 표시.
- 그런데 NOW 카드가 기본 문구 **"본 계약서를 준비 중입니다 / 준비되면 마이페이지에서 안내드립니다."** 에 멈춰 있음.
- **예식일·생년월일·주소를 입력하는 "계약서 요청" 카드(요청 항목)가 안 뜸.**

---

## 원인 — 같은 날 들어간 두 커밋이 서로를 몰랐음

마이페이지의 요청 카드는 백엔드가 내려주는 `contractInfo` 값이 있을 때만 렌더된다
(`mypage.html` → `renderContractRequest`). 그런데 백엔드 `buildContractInfoState`가
`contractInfo`를 **`null`로 반환**하고 있어서 카드가 통째로 사라졌다.

`null`로 빠진 이유는 6/8 같은 날 **24분 차이**로 들어간 두 변경이 충돌했기 때문이다.

| 시각 (KST) | 커밋 | 한 일 |
|---|---|---|
| 6/8 13:15 | `d752865` | **가입 시 `계약상태`를 `'미발송'`으로 초기화** (빈 값이 아님) |
| 6/8 13:39 | `491e7a3` | 계약서 전자서명 Phase 1 가드 추가 — **`계약상태`가 "비어있지 않으면" 발송된 것으로 판정** |

- 뒤 커밋(`491e7a3`)은 "계약서 발송 전이면 `계약상태`가 **비어 있다**"고 가정하고 가드를 짰다.
- 하지만 24분 앞선 커밋(`d752865`)이 이미 **모든 신규 가입자에게 `'미발송'`을 박아넣고** 있었다.
- 유효값은 `['미발송','발송','서명완료']`인데, 가드가 **기본값 `'미발송'`까지 `'발송'`과 똑같이 취급**해 버렸다.

그 결과 상담완료 단계에서:
- `buildContractInfoState`(70_journey.gs)가 `null` 반환 → **요청 폼이 사라짐**, NOW는 기본 '준비 중' 문구에 멈춤.
- 설령 폼이 떠도 `requestContract`(70_journey.gs)가 **"이미 계약이 진행 중입니다"** 로 제출까지 막던 상태.

### "전에는 나왔었는데" 의 정체
이 두 변경이 **GAS에 재배포되기 전**에는 `계약상태`가 비어 있어 폼이 정상으로 떴다.
재배포된 뒤부터 신규/갱신 행에 `'미발송'`이 박히면서 막힌 것. 즉 **코드 회귀(regression)** 다.

---

## 해결 — 가드가 `'미발송'`을 "미발송"으로 제대로 인식하게 수정

`automation/platform/70_journey.gs` **두 곳**에서, `'발송'` 또는 `'서명완료'`일 때만 차단하고
**`'미발송'`(가입 기본값·만료 후)은 요청 폼을 유지**하도록 변경했다.

### ① 표시 가드 — `buildContractInfoState`

```diff
- if (String(r.get('계약상태') || '').trim()) return null;   // 발송됨 → 계약 카드로
+ var _cs = String(r.get('계약상태') || '').trim();
+ if (_cs === '발송' || _cs === '서명완료') return null;   // 발송/서명완료 → 계약 카드로 ('미발송'=가입 기본값·만료 후 → 요청 폼 유지)
```

### ② 요청 제출 가드 — `requestContract`

```diff
- if (String(cust.get('계약상태') || '').trim()) return { ok: false, error: '이미 계약이 진행 중입니다.' };
+ var _cs = String(cust.get('계약상태') || '').trim();
+ if (_cs === '발송' || _cs === '서명완료') return { ok: false, error: '이미 계약이 진행 중입니다.' };   // '미발송'(가입 기본값·만료 후)은 재요청 허용
```

### 덤 — 만료 후 재요청도 정상화
계약서가 발송됐다가 72시간 만료되면 `expireUnsignedContracts`가 `계약상태`를 다시 `'미발송'`으로
되돌린다. 기존 가드에선 그 고객도 **영영 재요청을 못 했는데**, 이번 수정으로 만료 후 재요청도 자연히 풀린다.

> 시트의 `계약상태` 값은 손댈 필요 없다. **코드만 고치면 `'미발송'`이 정상 동작**한다.

## 배포

- 변경 파일: `automation/platform/70_journey.gs` (GAS 백엔드)
- `claude/hopeful-cray-h5e8ph` 브랜치에 커밋·푸시 완료.
- GAS `.gs`는 Vercel 배포 대상이 아니므로 `main` 푸시로는 반영되지 않음 → **GAS 재배포 필요.**

### 적용 절차
1. `70_journey.gs`를 GAS에 반영.
2. **`R3n9Mr` 배포 관리에서 "새 버전"으로 재배포** (저장만으론 `/exec`에 안 먹음).
3. 해당 고객 마이페이지 새로고침 → "예식일을 정하고 계약서를 요청하세요" 입력 카드 복귀 확인.
