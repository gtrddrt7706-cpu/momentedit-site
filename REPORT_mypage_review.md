## P0 — 먼저 고치면 좋은 것

### 1. 핀치 줌이 막혀 있어요 (접근성)
- **현상:** `<meta viewport ... maximum-scale=1.0, user-scalable=no>` (5번째 줄).
- **고객 영향:** 청첩장·계약·입금 화면을 **양가 부모님**도 봐요. 손가락 확대가 막히면 작은 글씨를 못 키워요. (WCAG 1.4.4 위반, iOS는 무시하지만 Android·태블릿은 실제로 막힘)
- **제안:** `maximum-scale`·`user-scalable=no` 제거 → `width=device-width, initial-scale=1, viewport-fit=cover`만. 입력창 자동 확대가 싫으면 input `font-size:16px` 이상으로 막는 게 정석(이미 15px라 거의 충분).

### 2. 내 개인코드가 마이페이지 어디에도 안 보여요
- **현상:** 코드카드 CSS(`.codecard`)와 복사 핸들러(`mp_copy`)는 있는데, **`mypageView` 본문에 `id="mp_code"` 마크업이 안 보임**(환영부엔 이름·상품 뱃지만). `renderMyPage`도 `$('mp_code')`를 가드(`if`)로만 처리.
- **고객 영향:** 메일 자동로그인으로 들어온 고객은 **자기 6자리 코드를 모를 수 있어요.** 나중에 메일 없이 직접 로그인하려면 코드가 필요한데, 마이페이지에서 확인할 길이 없으면 "코드 찾기"를 또 돌려야 해요.
- **제안:** 환영부(이름 아래) 또는 풋터 근처에 **개인코드 + 복사 버튼**을 다시 노출. (이미 `.codecard` 스타일이 있으니 마크업만 추가하면 끝)
- **확인 필요:** 의도적으로 숨긴 것인지(예: 보안)부터 확인 — 의도면 "코드 찾기"를 더 눈에 띄게.

## P1 — 경험을 또렷하게

### 3. 네트워크가 한 번 끊기면 로그아웃돼요
- **현상:** `loadMyState`의 `.catch` → 항상 `show('loginView')`. (silent 갱신만 예외)
- **고객 영향:** 지하철·엘리베이터에서 잠깐 끊겨도 **로그인 화면으로 튕겨요.** 캐시(직전 상태)가 있는데도 버려요.
- **제안:** 비-silent에서도 **catch 시 캐시가 있으면 캐시 화면 유지 + "연결이 잠시 불안정해요. 다시 시도" 토스트**. 토큰 만료(`ok:false`)일 때만 로그인으로.

### 4. 코드가 없는 방문자가 길을 잃어요
- **현상:** 로그인 화면 안내는 "개인코드/비번/메일 링크"만. **아직 신청 안 한 사람**을 위한 출구가 없음.
- **제안:** 로그인 화면 하단 링크줄에 **"아직 신청 전이신가요? → 상담 신청"**(`/` 또는 `/inquiry`) 한 줄 추가. 잘못 들어온 사람이 헤매지 않아요.

### 5. 작은 회색 글씨가 흐릿해요 (명도대비)
- **현상:** `--label-soft(#8A8475)`·`--light(#75705F)`를 11~12px 본문 보조(주의문구·NEXT 설명·카드 서브)에 광범위 사용. 밝은 배경(#FAFAF8)에서 작은 글씨는 대비 4.5:1에 못 미쳐요(특히 `--label-soft`는 ~3:1).
- **고객 영향:** 야외·고령·저휘도 화면에서 잘 안 읽혀요. 마침 **계약·입금·예약금** 같은 *중요한* 보조문구가 여기 많아요.
- **제안:** 12px 이하 중요한 안내는 `--sub(#5A554C)`로 한 단계 진하게, 순수 장식만 `--label-soft` 유지. (토큰 한두 곳 교체로 광역 반영)

### 6. 카톡이 비어 있으면 문의할 데가 없어요
- **현상:** 하단 문의 버튼은 `d.kakao` 있을 때만 노출, 없으면 숨김. 그 외 연락 수단은 화면에 없음.
- **제안:** 카톡 미설정 시 **이메일 문의(또는 안내 문구) 폴백**을 항상 한 줄. 고객이 막혔을 때 빠져나갈 곳은 늘 있어야 해요.

### 7. 날짜 표기를 화면 전체에서 통일
- **현상:** 완료 화면은 `2027. 12. 17` 식, 상담 카드는 서버 `prettyDate`, NEXT/안내는 또 다른 표현이 섞일 수 있음.
- **제안:** "YYYY. MM. DD (요일)" 한 포맷으로 통일(요일은 신뢰감↑). 한 곳(헬퍼)에서 포맷팅.

## P2 — 마감 디테일 (있으면 더 고급스러움)

### 8. 마이크로카피 톤
- "입금 확인 후 다음 단계 **해제**", "계약서 **발송**" 등 시스템 말투가 군데군데. → "입금이 확인되면 다음으로 넘어가요", "계약서를 보내드려요" 식 사람 말투로.
- 버튼이 영어(`Login`·`Send`)인데 부모님 세대엔 한글(`로그인`·`보내기`)이 더 친절. *브랜드 톤상 영어 유지*도 가능 — **결정 사항**.
- "STEP 3 / 7"·"NOW"·"NEXT"는 브랜드 감성으로 OK.

### 9. 비주얼 일관성
- **모서리 반경 혼재:** 카드 8 / 버튼·입력 3 / 코드복사 6 / 완료카드 10 / 모달 6. → 3·8·10 정도의 **반경 스케일**로 정리하면 더 정돈돼 보여요.
- **아이콘 언어:** 완료 화면엔 라인 아이콘, 다른 카드엔 아이콘 없음. → 각 여정 카드 제목 옆에 *아주 작은* 라인 아이콘을 통일해 넣으면 스캔성↑(과하면 역효과라 절제).
- **골드 토큰:** 코드에서 `--gold-bg`·`--gold-soft`를 쓰는데 `:root`엔 **정의가 없어요**(지금은 폴백값으로 동작). → `:root`에 정식 토큰으로 추가해 일관화(기술부채 정리).

### 10. 성능(폰트)
- Cormorant 5종 + Noto Sans KR 3종 + **Noto Serif KR 4종(300·400·500·600)** + Roboto Mono 2종을 한 번에 로드. 한글 세리프는 무거워요(서브셋이라도). → 실제 안 쓰는 굵기 정리(예: Serif 300/600 사용처 점검)로 첫 페인트 가속.

### 11. 작은 것들
- **NOW 헤드라인:** `nextAction`을 "첫 마침표"로 잘라 head/sub 분리 → 백엔드 문구에 마침표가 없으면 통문장이 head로(길어짐). 폴백 길이 가드 권장.
- **비밀번호 표시 토글(눈 아이콘)** 추가하면 입력 실수↓(부모님 대리 로그인 시).
- **모달 포커스:** 서명 모달 열릴 때 포커스가 모달로 이동/가둠이 없음(키보드·스크린리더). 여는 순간 첫 버튼에 `focus()` + Esc 닫기 정도.
- **완료 순간 딜라이트:** 청첩장 완성 화면 진입 시 골드 라인 그려지는 0.4s 모션 등 *절제된* 축하 한 번. (prefers-reduced-motion 존중)
- **터치 타깃:** 완료화면 [열기/공유/복사]가 좁은 폭에서 높이 ~36px → 44px 권장(파지 오타↓).

## 상태(빈/로딩/에러/예외) 점검
- **로딩:** 스피너+"불러오는 중…" 양호. 캐시 있으면 즉시 표시(이미 적용). ✔
- **빈 데이터:** 상담행 없을 때 카드 자동 숨김 — 에러 없이 렌더. ✔
- **예외(취소·노쇼·미계약):** `mp_exception` 뱃지로 분리 표시. ✔ — 다만 *왜* 그 상태인지/다음에 뭘 하면 되는지 **한 줄 안내**가 있으면 덜 불안해요(예: "예약이 취소됐어요 · 다시 신청하시려면 …").
- **강제 되돌리기:** 앞선 카드 숨김 + 데이터 보존(복구 안전). ✔ 설계 좋음.
