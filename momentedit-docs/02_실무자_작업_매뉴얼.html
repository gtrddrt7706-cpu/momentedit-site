Nº 02

계약 끝난 직후 신랑신부에게 카톡 보내기:

카톡 1차 안내안녕하세요 [신랑]님, [신부]님 :) Moment Edit 미쿠입니다. 청첩장 제작을 위해 몇 가지 정보를 받아두려고 해요. 한 번에 다 받지 않고, 며칠에 걸쳐 편하게 나눠서 보내주세요. 오늘 먼저 받을 정보는 이거예요: ✦ 신랑·신부 이름 (한글·영문 여권 표기) ✦ 결혼식 날짜·시간 (요일 포함) ✦ 결혼식 장소 (이름·주소) 언제든 편하게 답해주세요.
2
2~3일 후 · 부모님 정보

고객이 원하는 경우에만:

카톡 2차이번엔 부모님 정보를 받아두려고요. ✦ 신랑 측 아버님·어머님 성함 (혼주) ✦ 신부 측 아버님·어머님 성함 (혼주) ※ 청첩장에 부모님 성함 노출을 원치 않으시면 알려주세요. 혼주분 함자 없이도 청첩장은 만들 수 있어요.
3
3~5일 후 · 계좌 정보
카톡 3차이번엔 마음 전하기(계좌) 정보예요. ✦ 신랑 본인 계좌 (은행명 + 계좌번호) ✦ 신부 본인 계좌 (은행명 + 계좌번호) 부모님 계좌도 함께 받으시려면 알려주세요. 원치 않으시면 본인 계좌만 받습니다.

카톡 4차거의 다 왔어요! 마지막으로 청첩장 디자인을 골라주세요. 👉 https://momentedit.kr/invitation-gallery.html 8가지 디자인이 있어요. 01 Classic · 02 Editorial · 03 Letterpress · 04 Vermilion 05 Botanical · 06 Hangeul · 07 Architect · 08 Noir 마음에 드는 번호 알려주세요. (추천도 가능하니 편하게 말씀해주세요.)

  • 식장 주소 (필수) · 도로명 주소
  • 교통편 (필수) · "○호선 ○○역 ○번 출구 도보 ○분" 형식 권장

  • 가족 청첩장 디자인 (필수) · 8종 중 1개 선택 (라이브와 같거나 달라도 무관)

⚠️ 식장 전화번호는 시트에 입력해도 가족 카드 표시에는 사용되지 않음 (직계 가족은 직접 부부에게 문의함). 옵션으로만 기록.

Nº 03
청첩장 만들기라이브 + 가족 청첩장 동시 발행 (v4.0)
담당: huijun
흐름 (v4.0 옵션 C 반영) 라이브 청첩장은 AI 자동화 프롬프트로 HTML 생성 → GitHub /i/ 폴더 push.
가족 청첩장은 시트에 R~Z 9컬럼 추가 입력 → 메뉴 클릭만으로 자동 빌드 (별도 AI 작업 불필요).
📌 v4.0 변경 사항 이전 v3.x까지는 직계가족 청첩장이 별도 수작업이었으나, v4.0 옵션 C 구축 완료로 가족 청첩장도 시트 입력 → 메뉴 클릭만으로 자동 빌드됨. 본 패널은 라이브 청첩장 생성 흐름 중심으로 설명. 가족 청첩장은 다음 패널(P04 · 시트에 부부 등록)에서 R~Z 입력 후 자동 빌드.

규칙: 신랑이니셜3-신부이니셜3-월일

  • 예: 이민호 · 최유진 / 8월 23일 → lmh-cyj-0823
  • 예: 박서준 · 김지원 / 12월 5일 → psj-kjw-1205

영문 소문자 + 숫자 + 하이픈만 사용. 한글 절대 X.

⚠️ eventId는 라이브와 가족 청첩장 모두에 동일하게 사용됨. URL 라우팅·시트 PK·파일명 통일.

4
AI 응답 확인

AI는 다음 형식으로 응답:

  • ✅ 청첩장 생성 완료 알림
  • 수정 내역 (이름·날짜·eventId 등)
  • 검증 통과 항목 (test-couple 잔존 없음 등)
  • 완성된 HTML 전체 코드
5
HTML 코드 파일로 저장

AI가 준 HTML 코드 전체를 복사 → 메모장 또는 VSCode 열기 → 붙여넣기 → 파일로 저장:

  • 파일명: [eventId].html (예: lmh-cyj-0823.html)
  • 인코딩: UTF-8
  • 저장 위치: 우선 다운로드 폴더에 임시 저장
💡 검증 팁 AI가 응답한 검증 통과 목록을 꼭 확인. 특히 ?e=test-couple 잔존 검사·이전 부부 이름 잔존 검사가 통과되어야 함. 통과 안 됐다면 AI에게 재수정 요청.
🏗️ 가족 청첩장은 자동 빌드 (v4.0) 가족 청첩장은 별도 AI 작업 X. 다음 패널(P04)에서 Couples 시트 R~Z 9컬럼만 입력하고, 시트 메뉴 "Moment Edit → 가족 청첩장 빌드 (선택한 행)" 클릭하면 Apps Script가 자동으로 빌드 → GitHub /i-family/ 폴더에 push → Vercel 배포. 부부당 2분.

가족 청첩장 빌드 트리거 (v4.0 · R열 Y인 경우만) v4.0 신규

R~Z 입력이 끝났으면 시트 메뉴에서 바로 빌드:

  • 방금 입력한 부부 행의 아무 셀이나 클릭
  • 상단 메뉴 Moment Edit → 가족 청첩장 빌드 (선택한 행) 클릭
  • 약 5~10초 대기 → "빌드 완료" 알림창 표시 (eventId · 디자인 · URL)
  • BuildLogs 탭으로 이동 → 새 행에 status=OK 기록 확인
  • Vercel 자동 배포 1~2분 → URL 접속 가능

⚠️ 라이브 청첩장은 별도 시점에 GitHub /i/ 폴더에 수동 push (다음 패널 05). 가족은 자동.

⚠️ I열 weddingTime 주의 구글 시트가 "14:00" 입력 시 자동으로 시간 형식으로 바꿔버린다. 입력 전 반드시 I열 전체를 일반 텍스트 형식으로 변경:
  • I열 머리글(컬럼 "I") 클릭 → I열 전체 선택
  • 상단 메뉴: 서식 → 숫자 → 일반 텍스트
⚠️ S열 디자인 코드 주의 (v4.0) 01 그대로 입력하면 구글시트가 숫자 1로 변환 → 빌드 시 "잘못된 디자인 코드" 오류. 해결:
  • 방법 1: 앞에 작은따옴표 '01 입력 → 텍스트로 강제
  • 방법 2: S열 전체 선택 → 서식 → 숫자 → 일반 텍스트 → 값 재입력
⚠️ Z열 구글맵 iframe URL 추출법 (v4.0) iframe 코드 전체를 붙여넣으면 안 됨. src=""안의 URL만 추출해서 붙여넣기:
  • 구글맵(google.com/maps)에서 식장 검색
  • 식장 카드 → 공유 → "지도 퍼가기" 탭
  • <iframe src="여기부터...따옴표 직전까지" ...>
  • 예시: https://www.google.com/maps/embed?pb=!1m18!...
💡 부모 정보 P, Q열 고객이 부모 정보 노출을 원치 않으면 비워두면 됨. 라이브 페이지·가족 청첩장 모두에서 자동으로 해당 영역이 hide된다.
📊 BuildLogs 시트 활용 (v4.0) 가족 청첩장 빌드 결과는 자동으로 BuildLogs 시트에 기록됨. 오류 발생 시 error 컬럼에 상세 메시지 표시. 운영 중 문제 발견 시 BuildLogs 먼저 확인 → 트러블슈팅(P08)으로 분기.

  • 3초 이상 옛 정보 노출되면 → live.html FOUC 코드 누락 (AI에게 점검 요청)
증상 ④ · 편지 발송 안 됨
  • Couples 시트의 해당 eventId 행 D, E열(이메일) 정상 입력 확인
  • Banned 시트에 의도치 않은 금지어 포함 여부 확인
  • Apps Script 편집기 → testSendToBoth 함수 실행 → 정상 발송 확인
  • STUDIO_EMAIL이 huijun 계정 별칭으로 정확히 등록되어 있는지 확인
증상 ⑤ · 라이브 페이지에 빈 화면 (5초 이상)
  • 5초 후 failsafe 작동해서 더미 데이터 표시되어야 정상
  • ?e 파라미터의 eventId가 시트에 실제 등록되었는지 확인
  • Apps Script 새 배포가 활성 상태인지 확인
증상 ⑥ · 영상 안 나옴
  • Vimeo 영상이 실제 라이브 송출 중인지 확인
  • 시트 N열의 vimeoId가 숫자만 정확히 입력되었는지 확인
  • 비공개 영상이면 O열 vimeoHash 필수
  • 페이지 강력 새로고침 (Ctrl+Shift+R)
증상 ⑦ · 사이트 전체 안 열림
  • 가비아 도메인 만료 여부 확인 (2027.04.20)
  • Vercel 프로젝트 상태 (장애 페이지 표시되면 Vercel 자체 장애)
  • GitHub repository 비공개 전환 안 됐는지 확인 (반드시 Public 유지)
증상 ⑧ · 가족 청첩장 빌드 실패 - 디자인 코드 v4.0

BuildLogs status=ERROR + error="잘못된 디자인 코드: '1' (01~08 또는 01-classic 형식)"

  • 원인: 구글시트가 S열 01을 숫자 1로 자동 변환
  • 해결 1: S5 셀(또는 해당 행) 클릭 → 기존 값 삭제 → 작은따옴표 추가하여 '01 입력 → Enter
  • 해결 2 (영구): S열 전체 선택 → 메뉴 서식 → 숫자 → "일반 텍스트" → 값 재입력
  • 검증: 다시 빌드 트리거 → BuildLogs OK 확인
증상 ⑨ · 가족 청첩장 빌드 실패 - 필수 필드 누락 v4.0

BuildLogs error="필수 필드 누락: venueAddress, venueTransport, ..."

  • 원인: Couples 시트 R~Z 컬럼 중 필수 필드가 비어있음
  • 해결: 오류 메시지에 표시된 컬럼명 확인 → 시트에 값 입력 → 재빌드
  • 필수: familyDesign(S), venueNameKo(T), venueAddress(V), venueTransport(X), venueParking(Y), venueMapIframe(Z)
  • 옵션: venueNameEn(U), venueTel(W) — 비어있어도 빌드 진행
증상 ⑩ · 가족 청첩장 빌드 실패 - GitHub 인증 v4.0

BuildLogs error="GitHub push 실패 (401)" 또는 "GITHUB_TOKEN이 설정되지 않았습니다"

  • 원인 1: Apps Script Properties에 GITHUB_TOKEN 누락
  • 해결: Apps Script 편집기 → 프로젝트 설정(⚙️) → 스크립트 속성 → GITHUB_TOKEN 값 확인 · 추가
  • 원인 2: 토큰 만료 또는 폐기됨
  • 해결: github.com/settings/tokens → 새 토큰(classic · repo 권한) 발급 → Properties의 GITHUB_TOKEN 값 갱신
  • 연결 테스트: 시트 메뉴 "Moment Edit → 연결 테스트 (GitHub)" 클릭 → 성공 확인
증상 ⑪ · 가족 청첩장 구글맵 안 보임 v4.0
  • 원인: Z열 venueMapIframe URL 잘못 추출 (iframe 태그 전체 붙여넣기 등)
  • 해결: Z열 값 확인 → https://www.google.com/maps/embed?pb=... 형태인지 확인 → src=""안의 URL만 추출해서 재입력 → 재빌드
  • 가족 페이지에 404 화면이 보이면 URL이 잘못 추출된 것
증상 ⑫ · 보안 사고 - 토큰 노출 v4.0

실수로 GitHub PAT(ghp_...)를 채팅·스크린샷·코드 파일 등에 노출한 경우

  1. 즉시 폐기 · github.com/settings/tokens → 노출된 토큰 옆 빨간 Delete 버튼
  2. 새 토큰 발급 · Generate new token (classic) → repo 권한 · No expiration → 토큰 복사 (메모장에만 임시)
  3. Properties 갱신 · Apps Script 편집기 → 프로젝트 설정 → GITHUB_TOKEN 값 새 토큰으로 교체 → 저장
  4. 연결 테스트 · 시트 메뉴 "Moment Edit → 연결 테스트 (GitHub)" → 성공 확인
  5. 메모장 토큰 삭제 · 임시 저장한 토큰 정리
🚨 긴급 롤백 절차
  1. GitHub Desktop → History 탭 → 마지막 안전 커밋 우클릭 → "Revert this commit"
  2. Push origin → Vercel 자동 재배포 (1~2분)
  3. 또는 백업 파일(live_v3.1_backup_YYYYMMDD.html) 직접 복원
  4. Apps Script는 배포 관리 → 보관처리된 이전 버전 활성화로 롤백
  5. 가족 청첩장 롤백 (v4.0): 시트 데이터 수정 → 빌드 트리거 재실행. 또는 GitHub에서 i-family/[eventId].html 이전 커밋으로 revert.
💡 AI에게 도움 요청 시 (v4.0 짝 문서 동기화) 두 파일 모두 함께 첨부:
  • 01_AI_시스템_마스터.html v5.0 (시스템 컨텍스트)
  • 02_실무자_매뉴얼.html v4.0 (본 매뉴얼)
그리고 문제 상황 캡처도 함께 전달. 두 파일을 짝으로 보내면 AI가 시스템 컨텍스트 + 실무 흐름을 모두 이해해서 정확한 해결책을 제시할 수 있음. 가족 청첩장 문제는 BuildLogs 시트 스크린샷도 함께 첨부하면 빠른 진단 가능.
