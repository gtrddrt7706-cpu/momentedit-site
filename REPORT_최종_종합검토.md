# 최종 종합 검토 — 전체 제품 관점 (마지막 패스)

**범위:** 그동안은 *깊이*(마이페이지·관리자·아키텍처)를 봤고, 이번엔 *폭* — 랜딩→상담신청→마이페이지 **퍼널**, 공개 페이지, 측정, 법, 끝단(이메일/링크).
**결론 먼저:** **새로 발견된 '큰 구멍'은 없음.** 제품은 디테일·구조 모두 탄탄. 남은 건 *이미 정리된 로드맵 실행* + 아래 **폭 항목 3개**만 합치면 검토 종료.

---

## A. 확인되어 '안심해도 되는' 것 (실제로 잘 돼 있음)
- **개인정보 수집·이용 동의** 체크박스(inquiry `#privacyConsent` + 검증) ✓ · `privacy.html` 존재 ✓ · **허니팟 봇차단** ✓ → 법·스팸 기본기 OK
- **공유/검색 메타**: og·twitter·description·robots·sitemap 전 공개페이지 ✓
- **관리자 안전장치**: 확인 모달·위험액션 danger·이력·강제변경+사유 ✓
- **아키텍처**: 정적+GAS+Sheets는 규모에 적정(과설계 아님) ✓
- **전환 동선**: index에 가격(시그니처 210/280만)·문의·예약 CTA 노출 ✓

> 즉, "기본이 안 돼 있어서 위험한" 항목은 없음.

## B. 이번에 발견한 '폭' 개선점 (신규)

### B1. 측정(애널리틱스)이 **전 페이지 0** — 비즈니스 맹점 [높음]
- GA/gtag/픽셀/Plausible **전무**. → 방문수·**index→inquiry 전환율**·유입 경로(검색/카톡/광고)를 **전혀 못 봄**. 마케팅·콘텐츠 판단 근거가 0.
- **제안:** GA4(gtag) 1개 또는 가벼운 Plausible를 전 페이지에 1줄. 핵심 이벤트만: *상담신청 제출 · 일정 신청 · 청첩장 발행*. 프런트만·즉시·저비용 고효용.

### B2. 청첩장 입력이 아직 **구글 폼** [중·구조]
- `form.html` = 외부 **Google Forms로 redirect**. → ① **브랜드 경험 이탈**(구글 폼 디자인·로고) ② 우리가 만든 **마이페이지 위저드와 이중 경로**(같은 Couples를 두 곳에서 기록 = 드리프트 위험, 아키텍처 C2와 동일).
- **제안:** 장기 — 마이페이지 위저드로 **단일화**(구글 폼 폐기). 단기 — 최소한 진입을 브랜드 페이지로 감싸고 안내 일관화.

### B3. 핀치 줌 차단 **4페이지**(inquiry 포함) [중·접근성]
- `user-scalable=no`: inquiry·mypage·admin·schedule. 특히 **inquiry는 전환 폼** — 부모님·중장년이 작은 글씨 못 키우면 이탈↑. (index·live는 줌 허용 — 일관성도 깨짐)
- **제안:** 전 페이지에서 `user-scalable=no`·`maximum-scale` 제거(입력 자동확대는 16px로 방지).

### B4. 끝단(완료 이메일 ↔ 인앱 완료화면) 톤·정보 불일치 [중]
- 완료 메일(첨부2)은 *구글 폼 경로* 산출 / 마이페이지 발행은 *인앱 완료화면*(메일 없음). 같은 "청첩장 완성"인데 **두 끝단의 모양·정보가 다름**.
- **제안:** 마이페이지 발행에도 완료 메일을 보낼지 결정(잔금 PLAN의 결정과 함께), 보낸다면 완료화면과 **동일 톤**으로.

### (참고) inquiry 폼 길이 [낮음]
- 체크박스 17·입력 9·텍스트영역 2로 *고민 많은* 상담 신청서. 프리미엄 스튜디오엔 적정(리드 선별)이나, **진행률 표시/선택 섹션 접기**로 이탈을 더 줄일 수 있음.

## C. 이미 큐에 있는 핵심 (재확인 — 중복 안 함)
- 관리자 **다이닝·식순 스테일**(2차 B1) · **잔금**(PLAN) · 프런트 디테일 **배치 A** · **shared 공통화 연결** · **clasp**(준비완료).

## D. "내가 사장이라면" — 우선순위 Top 5
1. **clasp 셋업** — 반복 고통·드리프트 제거(준비 끝, 너 1회).
2. **관리자 다이닝·식순 표시** — 출시한 기능의 운영 공백.
3. **잔금 플로우** — 매출 마무리 단계 누락.
4. **애널리틱스 1개(B1)** — 비즈니스가 눈을 뜸(저비용·고효용).
5. **배치 A 디테일**(핀치줌·네트워크 회복력·연락 폴백) — 전환·신뢰.

## 결론
탐색은 여기서 **사실상 종료** — 더 찾을 큰 구멍 없음. 신규 폭 항목(측정·구글폼·핀치줌·끝단)을 로드맵 **배치 E**로 합쳐 두었고, 이제 남은 일은 *실행*뿐. "무엇부터" 정해주면 그 배치를 바로 돌릴게요.
