## 1. 핵심 구조

- **단계 머신(현재단계) 불변:** `… 입금완료 → 제작중 → 예식완료 → 결과물전달`
  - 잔금은 **단계가 아님** → 새 단계 추가 X, 단계 전이 트리거 X.
- **잔금 = 독립 상태축(`잔금상태`):** `대기 → 완료신호 → 확인` (계약금 `입금상태`와 동일 패턴).
  - 제작중·예식완료 동안 **마이페이지에 잔금 카드**로 병행 노출.
  - 예식일 기준 **D-7 마감**, 그 전엔 "미리 입금 가능", D-7부터 "이제 입금" 강조 + 리마인드.
- **제작 3트랙(청첩장·다이닝·식순):** 잔금과 무관하게 **계속 수정 가능**(지금 그대로). "완료 ✓"는 체크리스트.
- **금액:** 이미 있는 `_journeyAmounts(계약총액)`의 `잔금`(=총액 80%) 재사용. 새 계산 없음.

## 3. 잔금 상태 머신

```
대기(기본)
  └─(제작 단계 진입 + 계약 서명완료)→ 카드 노출: 금액·계좌·"미리 입금 가능"
        └─ 고객 [입금했어요] ─→ 완료신호  (잔금입금자명·잔금입금신호 기록)
                                   └─ 관리자 통장 대조 [잔금 입금확인] ─→ 확인 (잔금확인일시)
예식일 D-7 도달(상태 무관):
  - 카드 강조 + 리마인드 메일(+카톡 링크) 1회  (잔금리마인드 기록 → 중복 방지)
확인 이후: 마이페이지 잔금 카드 = 접힘(완료) / NEXT에서 제거
```

- **노출 조건(buildBalanceState):** `계약상태=서명완료` && `현재단계 ∈ {입금완료, 제작중, 예식완료}` && (`잔금상태≠확인` 또는 최근 완료 접이식).
- **게이트(handleBalanceSignal):** 위 단계 + `잔금상태≠확인`일 때만 신호 허용. 멱등.

## 5. 안전장치 / 엣지

- **제작 잠금 절대 없음:** 잔금 상태가 무엇이든 `PRODUCTION_STAGES`(입금완료·제작중)에서 제작 편집 그대로. 잔금은 단계를 안 바꾸므로 영향 0.
- **미리 입금:** D-7 전에도 [입금했어요] 가능 → `완료신호` → 관리자 확인. 정상 흐름.
- **중복 리마인드 방지:** `잔금리마인드` 플래그로 1회만.
- **계약총액 없음:** `_journeyAmounts`=null → 카드는 "금액은 디렉터 확인 후 안내"(계약금 카드와 동일 폴백).
- **멱등:** 신호·확인 모두 이미 처리 시 `{ok:true, already:true}`.
- **예식일 누락:** 제작 기초정보 미입력이면 `예식일` 없음 → 잔금 카드는 "예식일 확정 후 안내"로 표시, D-7 트리거 대상 제외.
