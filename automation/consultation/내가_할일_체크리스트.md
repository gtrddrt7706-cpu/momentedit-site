# 내가 직접 할 일 (운영자 체크리스트)

코드 작성은 끝났습니다. 아래는 **연결·입력·배포·테스트**뿐입니다.

## 0. 먼저 준비할 정보 (메모해 두기)
- [ ] 미쿠(관리자) 이메일 — 알림 받을 주소
- [ ] 입금 **계좌번호 + 예금주**
- [ ] 스튜디오 **정확한 도로명 주소** (확정 메일에만 노출)
- [ ] **카카오톡 채널 URL**
- [ ] 상담 1회 길이(분), 하루 슬롯 **시각 3개** (예: 11:30 / 14:50 / 18:10)

## 1. 시트 + Apps Script 만들기
- [ ] 구글 드라이브에 **새 구글 시트** 생성
- [ ] 시트 메뉴 **확장 프로그램 → Apps Script**
- [ ] `consultation-booking.gs` 내용을 기본 `Code.gs`에 **전체 붙여넣기**
- [ ] 좌측 **＋ → HTML** 로 3개 만들기 — 이름은 `.html` 빼고 정확히
      `ScreenA_apply` / `ScreenB_schedule` / `ScreenC_change`

## 2. 상담용 캘린더 준비
- [ ] 구글 캘린더에서 **새 캘린더** 생성 → 설정에서 **캘린더 ID 복사**
- [ ] 상담 받을 날에 제목에 **`상담가능`** 이 들어간 일정 만들기 (이 날만 달력에 열림)
- [ ] (예식 등 다른 일정은 그 시간 슬롯을 자동 "마감" 처리)

## 3. 설치 실행 (권한 허용)
- [ ] 함수 선택을 **`setupConsultation`** 으로 → **실행**
- [ ] 권한 동의창 → 계정 선택 → 고급 → 허용 (시트·Gmail·캘린더)
- [ ] 로그에 "✅ setup 완료" 확인 (시트에 `상담예약` 탭 생김)

## 4. CONFIG 값 채우기  ← 지금까지 비워둔 부분
- [ ] `.gs` 상단 `CONFIG`의 `[...]` 6개를 실제 값으로:
      계좌·예금주·주소·카톡·미쿠이메일·캘린더ID
- [ ] `SLOTS`·`SLOT_DURATION_MIN`도 확정값으로

## 5. 웹앱 배포
- [ ] **배포 → 새 배포 → 웹 앱**
- [ ] **실행: 나(Me) / 액세스: 모든 사용자(Anyone)** ← 꼭 이렇게
- [ ] 나온 **웹앱 URL**을 홈페이지 "대면 상담 신청" 버튼에 연결
- [ ] (코드 수정 후엔 **배포 관리 → 새 버전**으로 갱신해야 반영됨)

## 6. 오픈 전 테스트
- [ ] 캘린더에 `상담가능` 1~2개 → 웹앱 URL로 본인 신청
- [ ] 메일①(전용 URL) → 화면 B에서 이름·가능일·슬롯 확인 → 신청
- [ ] 미쿠 메일②의 **[승인]** → 고객 확정 메일③ + 캘린더 일정 생성 확인
- [ ] **[변경 제안]** → 화면 C 입력 → 고객 메일④ → **[수락]** → 변경 확정⑤ 확인
- [ ] 확정된 슬롯이 화면 B에서 **"마감"** 표시 + 클릭 불가 확인

> 꼭: 스크립트·시트·캘린더 **타임존을 Asia/Seoul**로 (날짜/시간 매칭에 중요).
