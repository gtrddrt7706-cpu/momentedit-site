# Moment Edit · 대면 상담 예약 시스템 — 운영자 설정 가이드

> 코드는 모두 만들어져 있습니다. 아래 순서대로 **연결·입력·배포·테스트**만 하시면 됩니다.
> CONFIG의 `[...]` 값은 아직 비어 있습니다 — 6단계에서 직접 채우시면 됩니다.

## 파일 구성 (이 폴더)
| 파일 | Apps Script 안에서의 이름 | 역할 |
|---|---|---|
| `consultation-booking.gs` | (스크립트 파일) | 백엔드 전체 — CONFIG·라우팅·토큰·시트·캘린더·메일 5종·onEdit·설치 |
| `ScreenA_apply.html` | `ScreenA_apply` (HTML) | 화면 A — 상담 신청 폼 (공개, 홈페이지) |
| `ScreenB_schedule.html` | `ScreenB_schedule` (HTML) | 화면 B — 스케줄 선택 (디자인 확정본 그대로 + 실연동) |
| `ScreenC_change.html` | `ScreenC_change` (HTML) | 화면 C — 미쿠 변경입력 미니페이지 |

> ⚠️ Apps Script 편집기에서 HTML 파일을 만들 때 **이름에 `.html`을 붙이지 마세요.** 파일명은
> 정확히 `ScreenA_apply` / `ScreenB_schedule` / `ScreenC_change` 여야 합니다 (.gs가 이 이름으로 불러옵니다).

---

## STEP 1 · 구글 시트 + Apps Script 만들기
1. 구글 드라이브에서 **새 구글 시트** 생성 (예: "Moment Edit 상담예약").
2. 상단 메뉴 **확장 프로그램 → Apps Script**.
3. 기본 `Code.gs`에 `consultation-booking.gs` 내용을 **전체 붙여넣기**.
4. 좌측 파일 **＋ → HTML** 로 3개 파일 생성, 각각 위 표의 이름으로 만들고 내용 붙여넣기.

## STEP 2 · 상담용 구글 캘린더 준비
1. 구글 캘린더에서 **새 캘린더**를 하나 만듭니다 (예: "상담 가능 시간").
2. 그 캘린더 **설정 → 캘린더 통합 → 캘린더 ID** 를 복사해 둡니다 (6단계에서 입력).
3. **가능일 표시법**: 상담을 받을 날·시간에 제목에 **`상담가능`** 이 들어간 일정을 만듭니다.
   - 예: "상담가능" 종일 일정을 6/3, 6/5 에 → 그 날들이 달력에서 활성화됩니다.
   - **예식 등 다른 일정**(제목에 `상담가능` 없음)은 자동으로 그 시간 슬롯을 "마감" 처리하고,
     종일이면 그 날 전체를 가능일에서 제외합니다.

## STEP 3 · 미쿠 이메일 / 계좌 / 주소 / 카톡 준비
6단계에서 입력할 값들을 미리 확정해 둡니다.
- 미쿠(관리자) 이메일 — 알림 메일을 받을 주소
- 입금 계좌 + 예금주
- 정확한 도로명 주소 (확정 메일에만 노출)
- 카카오톡 채널 URL
- 상담 1회 길이(분)와 하루 슬롯 시각 3개 (예: 11:30 / 14:50 / 18:10)

## STEP 4 · 입금 방식
- 본 시스템은 PG 결제 연동이 없습니다. **계좌이체 후 미쿠 승인** 방식입니다.
- 고객은 화면 B에서 시간 선택 시 계좌 안내를 보고 입금합니다.
- 미쿠는 입금을 직접 확인한 뒤, 알림 메일의 **[✓ 승인하기]** 를 눌러 확정합니다.

## STEP 5 · 권한 승인
1. Apps Script 편집기에서 함수 목록을 **`setupConsultation`** 으로 선택 → **실행**.
2. 처음 실행 시 권한 동의 창이 뜹니다 → 계정 선택 → 고급 → 허용
   (시트·Gmail·캘린더 권한이 필요합니다).
3. 실행 로그에 "✅ setup 완료" 가 보이면, 시트에 **`상담예약`** 탭과 헤더가 생성됩니다.
   - onEdit 보조 트리거, 버튼 서명용 비밀키도 이때 자동 생성됩니다.

## STEP 6 · CONFIG 값 채우기  ← 직접 입력
`consultation-booking.gs` 상단 `CONFIG` 의 `[...]` 부분을 실제 값으로 바꿉니다.
```js
SLOT_DURATION_MIN: 40,                 // 상담 길이(분) 확정값
SLOTS: ['11:30','14:50','18:10'],      // 하루 슬롯 3개 확정값
ACCOUNT: '하나은행 123-456-789012',     // ← 실제 계좌
ACCOUNT_HOLDER: '홍길동',                // ← 예금주
STUDIO_ADDRESS: '경기 고양시 …',         // ← 정확한 도로명 주소
KAKAO_URL: 'https://pf.kakao.com/...',  // ← 카카오톡 채널
ADMIN_EMAIL: 'miku@example.com',         // ← 미쿠 이메일
CALENDAR_ID: 'xxxxx@group.calendar.google.com', // ← 2단계 캘린더 ID
```
> DEPOSIT(예약금)·URL_VALID_DAYS·CONFIRM_DEADLINE_HOURS 는 기본값이 들어 있으니 필요할 때만 바꾸세요.

## STEP 7 · 웹앱 배포
1. 우측 상단 **배포 → 새 배포** → 유형 **웹 앱**.
2. 설정:
   - **실행 계정: 나(Me)**
   - **액세스 권한: 모든 사용자(Anyone)**  ← 고객이 로그인 없이 접속해야 하므로 필수
3. 배포하면 **웹 앱 URL** 이 나옵니다. 이 URL이 **화면 A(상담 신청 폼)** 주소입니다.
   - 홈페이지의 "대면 상담 신청" 버튼을 이 URL로 연결하거나 iframe 으로 임베드하세요.
   - 화면 B/C 는 시스템이 토큰을 붙여 자동으로 안내하므로 따로 링크할 필요가 없습니다.
> 코드를 수정한 뒤에는 **배포 → 배포 관리 → 편집(연필) → 버전: 새 버전 → 배포** 로 갱신해야 반영됩니다.

## ✅ 테스트 절차 (오픈 전 1회)
1. 캘린더에 가까운 날 **`상담가능`** 일정을 1~2개 만든다.
2. 웹앱 URL(화면 A)에 접속 → 본인 정보로 신청 → **메일①(전용 URL)** 수신 확인.
3. 메일①의 버튼 → 화면 B → 이름이 보이고, `상담가능` 날만 활성화되는지 확인 → 시간 선택 → 신청.
4. 미쿠 메일로 **메일②(알림 + [승인]/[변경제안] 버튼)** 수신 확인.
5. **[✓ 승인하기]** → 고객에게 **메일③(예약 확정 + 주소)** 도착 + 캘린더에 일정 생성 확인.
6. 다른 신청으로 **[시간 변경 제안]** → 화면 C에서 새 시간 입력 → 고객 **메일④** → **[수락]** → **메일⑤(변경 확정)** 확인.
7. 같은 전용 URL 재방문(셀프 변경) → 다른 시간 재선택 동작 확인.
8. 이미 확정된 슬롯이 화면 B에서 **"마감"** 으로 보이고 클릭이 안 되는지 확인.

## 운영 중 미쿠가 하는 일은 딱 2가지
- ① 구글 캘린더에 `상담가능` 일정 올리기 (가끔)
- ② 입금 확인 후, 알림 메일의 **[승인]** 또는 **[변경 제안]** 버튼 누르기 (예약 1건당)

그 외(예약 기록·달력 갱신·슬롯 마감·메일 발송)는 전부 자동입니다. 시트를 직접 편집할 필요가 없습니다.

## 참고 · 동작 메모
- **스케줄 비공개**: 공개 시간표가 없습니다. 화면 B는 전용 토큰 URL로만 열립니다.
- **신청 ≠ 확정**: 화면·메일 모두 "확정 메일을 받아야 예약 완료"임을 명시합니다. 메일①은 "신청 접수",
  메일③·⑤만 "확정" 톤입니다.
- **보안**: 승인/변경 버튼 링크에는 서명(sig)이 들어 있어, 고객 토큰만으로는 승인할 수 없습니다.
- **만료**: 신청 후 `URL_VALID_DAYS`(기본 7일)가 지난 전용 URL은 자동으로 안내 페이지를 보여줍니다.
- **타임존**: 스크립트/시트/캘린더는 모두 **Asia/Seoul** 로 맞춰 주세요 (파일 → 프로젝트 설정 / 시트 설정).
