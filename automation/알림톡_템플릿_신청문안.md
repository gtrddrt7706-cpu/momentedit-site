## 0. 사전 준비 (순서대로)

1. **카카오톡 채널** 개설 + **비즈니스 채널 전환**(사업자등록증 인증) — https://center-pf.kakao.com
2. **솔라피(solapi.com) 가입** → 콘솔에서:
   - ① 발신번호 등록(통신서비스 이용증명원 또는 휴대폰 인증) → `SOLAPI_SENDER`
   - ② 카카오 채널 연동(카카오 비즈니스 > 발신프로필) → 발급된 **pfId** → `SOLAPI_PF_ID`
   - ③ API Key·Secret 발급 → `SOLAPI_API_KEY` / `SOLAPI_API_SECRET`
   - ④ 충전(선불) — 알림톡 1건 약 9원, SMS 약 20원, LMS 약 50원 수준
3. 아래 **템플릿 16종**을 솔라피 콘솔 > 카카오 > 템플릿에서 등록 → 카카오 검수(영업일 1~3일)
4. 승인된 템플릿의 **템플릿 ID(KA01TP…)** 를 GAS Script Properties `KAKAO_TEMPLATES`에 JSON으로 입력

### GAS Script Properties (프로젝트 설정 > 스크립트 속성)

| 속성 | 값 | 비고 |
|---|---|---|
| `NOTIFY_ENABLED` | `true` | 이걸 넣는 순간 실발송 시작(그 전엔 로그만) |
| `SOLAPI_API_KEY` | 콘솔에서 복사 | |
| `SOLAPI_API_SECRET` | 콘솔에서 복사 | |
| `SOLAPI_SENDER` | `01000000000` | 등록한 발신번호, 숫자만 |
| `SOLAPI_PF_ID` | 채널 연동 후 발급 | 없으면 알림톡 생략, 전부 SMS |
| `ADMIN_PHONE` | 디렉터 휴대폰, 숫자만 | 관리자 알림(SMS) 수신처 |
| `KAKAO_TEMPLATES` | 아래 "승인 후 매핑" 참고 | 승인된 것만 채우면 됨 |

## 2. 승인 후 매핑 — `KAKAO_TEMPLATES`

승인된 템플릿 ID(KA01TP…)를 이벤트 키에 매핑한 JSON **한 줄**을 Script Properties에 넣습니다.
공용 템플릿(T08·T09)은 두 이벤트에 같은 ID를 넣으면 됩니다. **승인된 것만 넣으면 되고, 빠진 이벤트는 자동으로 SMS 발송.**

```json
{"cust.consultConfirmed":"KA01TP...","cust.consultDayBefore":"KA01TP...","cust.timeProposed":"KA01TP...","cust.fittingRequest":"KA01TP...","cust.contractArrived":"KA01TP...","cust.paymentConfirmed":"KA01TP...","cust.cashReceiptIssued":"KA01TP...","cust.midPre":"KA01TP같은ID","cust.midDue":"KA01TP같은ID","cust.balancePre":"KA01TP같은ID2","cust.balanceDue":"KA01TP같은ID2","cust.resultDelivered":"KA01TP...","cust.holdGranted":"KA01TP...","cust.holdReleased":"KA01TP...","cust.holdExpiring":"KA01TP...","cust.changeConfirmed":"KA01TP...","cust.changeDeclined":"KA01TP...","cust.archiveExpiring":"KA01TP..."}
```

## 3. 테스트 순서 (GAS 편집기에서)

1. `notifySetupCheck()` 실행 → 로그에서 설정 누락 확인
2. `notifyTestAdminSms()` 실행 → 디렉터 폰에 문자 오면 SMS 연동 성공
3. 테스트 고객(본인 번호로 만든 코드)으로 `notifyTestCustomerByCode('코드')` → 알림톡(또는 SMS) 수신 확인
4. 이후 실제 흐름(상담 확정 등)에서 자동 발송 — 실행 로그에 `[notify] 발송 OK`가 남음

※ 관리자용 알림(입금신호·변경요청 등 14종)은 템플릿 승인 없이 전부 SMS로 나갑니다.
※ 발송 실패는 본 흐름(계약·입금 처리)을 절대 막지 않고 로그만 남깁니다.
