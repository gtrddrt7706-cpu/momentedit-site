## 1. 템플릿 16종 v2 (콘솔에 그대로 복붙)

### T01 · 상담 확정 — `cust.consultConfirmed`
심리: 결정 직후 안심+단독 예우(한 타임 한 팀을 사실로 전달) → 노쇼 억제·기대 상승

· 일시: #{일시}

이 시간은 두 분만을 위해 비워두는 단독 상담입니다. 일정 변경이 필요하시면 마이페이지에서 미리 알려주세요. 편안한 마음으로 오시면 됩니다.
```
버튼: `마이페이지 열기`

### T02 · 상담 하루 전 — `cust.consultDayBefore`
심리: 부담 제거(준비물 없음)+환대 → 출석률·첫인상
```
#{이름}님, 내일 모먼트에디트 상담이 예정되어 있어요.

· 일시: #{일시}

준비물은 없습니다. 두 분의 이야기를 들을 준비는 저희가 해두겠습니다. 내일 뵙겠습니다.
```
버튼: 없음

### T03 · 상담 시간 변경 제안 — `cust.timeProposed`
심리: 주도권 돌려주기(수락/다른 시간)+빠른 응답의 이득 명시 → 일정 확정률
```

· 제안 일시: #{일시}

마이페이지에서 수락하시거나 더 편한 시간을 선택하실 수 있어요. 확인해 주시면 그 시간으로 바로 확정해 드릴게요.
```
버튼: `마이페이지에서 확인`

### T04 · 시착 동의서 도착 — `cust.fittingRequest`
심리: 절차 투명성=신뢰(문서 목적을 먼저 설명) → 서명 거부감 완화
```
#{이름}님, 드레스 시착 동의서가 도착했어요.

시착 진행 기준과 비용 안내를 담은 문서입니다. 마이페이지에서 내용을 확인하신 뒤 서명해 주세요. 서명 후 시착이 진행됩니다.
```
버튼: `동의서 확인·서명`

### T05 · 계약서 도착 — `cust.contractArrived`
심리: 전환의 순간. 일관성 신호("상담에서 나눈 그대로")+차분한 검토 권유, 기한은 사실로 명확히 → 서명 전환
```

## 3. 테스트 순서 (GAS 편집기에서)

1. `95_notify` 파일 열고 → `notifySetupCheck()` 실행 → 로그에서 설정 누락 확인
2. `95_notify` 파일 → `notifyTestAdminSms()` 실행 → 디렉터 폰에 문자 오면 SMS 연동 성공
3. 테스트 고객(본인 번호로 만든 코드)으로 `notifyTestCustomerByCode('코드')` → 알림톡(또는 SMS) 수신 확인
4. 이후 실제 흐름(상담 확정 등)에서 자동 발송 — 실행 로그에 `[notify] 발송 OK`가 남음

※ 관리자용 알림(입금신호·변경요청 등)은 템플릿 승인 없이 전부 SMS로 나갑니다.
※ 발송 실패는 본 흐름(계약·입금 처리)을 절대 막지 않고 로그만 남깁니다.
※ 발송 시각: 21시~익일 8시 사이 고객 알림은 자동 보류 후 아침 8시에 발송(야간 방해 금지 설계).
