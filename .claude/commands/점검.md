# Moment Edit 통합 점검

너는 지금 momentedit.kr(Vercel 정적 HTML + GAS 백엔드 `R3n9Mr`)의 품질 점검관이다.
요청 범위: **$ARGUMENTS** (비어 있으면 `최근커밋` = 최근 변경분 중심으로 본다.)

## 0. 시작 전 (항상)
- `git log --oneline -20` 으로 최근 변경 파악. 범위가 `최근커밋`이면 마지막 점검 이후·최근 15~20커밋만 깊게 본다. 범위가 특정 기능/페이지면 거기에 집중하되 연동 지점은 따라간다.
- **전제 함정 주의**: 배포 기준은 항상 `origin/main`. 로컬·다른 브랜치가 최신이라 가정하지 말 것.
- 자동 점검 먼저 돌려 바닥을 깐다(둘 다 repo에 있음):
  - `node scripts/audit/gas-lint.mjs` — .gs 로드·구문·전역참조 이상
  - `node scripts/audit/render-check.mjs` — 인라인 스크립트 구문 + mypage·admin 실제 로드 pageerror (puppeteer 없으면 `npm i puppeteer`, 또는 `/tmp/dz` 하네스 재사용)
- 화면 판정은 **반드시 렌더로 확인**한다(코드만 읽고 단정 금지). puppeteer로 localhost 띄워 실제 DOM·텍스트를 본다. script.google.com 응답 목에는 항상 `Access-Control-Allow-Origin:'*'` 포함.

## 1. 점검 영역 (모든 것을 본다 가정 — 빠짐없이)
1. **시스템** — 로드/빌드/구문, env(키)·배포 상태, 콘솔/pageerror, 스크롤바·반응형·빈상태·로딩.
2. **기능** — 각 단계 버튼·폼·위저드가 의도대로 동작하는가(정상 경로 + 실패 경로).
3. **연동** — 프론트 ↔ GAS(api), ↔ 구글 캘린더, ↔ 메일/카톡 알림, ↔ Vercel AI(advisor·dining-match), 슬롯/예식일 단일 출처 정합.
4. **버그** — 실제 오류·깨짐. NaN/undefined/null 노출, 죽은 코드가 화면에 새는지.
5. **고객 입장** — 첫 방문~결과물까지 여정 자연스러움, 문구 친절·오해 없음, 다음 행동이 명확한가.
6. **관리자 입장** — 처리할 일 큐가 직관적인가, 오조작 위험, 필요한 정보가 클릭 없이 보이는가.
7. **모순** — 화면 간·문구 간 상충(예: 한 곳은 시간 표시, 다른 곳은 누락 / 완료인데 또 안내).
8. **계약서 vs 사실** — 계약서 본문(조항·금액·일시·기간)이 실제 시스템 동작·정책과 일치하는가(예식 일시·총액·D-149/D-9·위약 구간·보관 6개월).
9. **계산식** — 금액(계약금·중도금·잔금·예약금 충당), 위약금 구간, D-day, 현금영수증 발급 대상·기한, 환불 예상.

### + 추가로 항상 보는 것 (α)
10. **보안·권한** — _requireAdmin·토큰 검증, 개인정보·API키 노출 0, 출력 escape(XSS), CORS.
11. **문구·표기 일관성** — 전각 줄표(—) 금지(`·`/문장분리), **날짜·시간은 한 몸**, 호칭·톤 일관.
12. **동시성·멱등** — 이중 클릭·동시 제출·LockService, 재배포 갭(구 GAS 폴백) 안전.
13. **엣지·예외 입력** — 빈값·극단값·과거날짜·특수문자. 필요하면 퍼즈(수백 회) 돌려 0 에러 확인.
14. **상태 전이·여정** — 단계 흐름·되돌리기·취소/노쇼/미계약 분기·만료 처리.
15. **알림 트리거** — 발송 조건·야간 보류·중복 발송 없음.
16. **성능** — 콜드스타트·캐시 워밍·로딩 피드백·대용량.
17. **데이터 정합·영속** — 동의기록 read-modify-write 키 보존, 헤더(컬럼) 드리프트.
18. **법·규정** — 현금영수증 의무발급, 소비자분쟁해결기준 위약 구간, 개인정보처리방침·청약철회.
19. **회귀** — 최근 변경이 기존 기능을 깨지 않았는가(인접 기능까지 확인).

## 2. 발견 시 행동
- **즉시 수정**한다(읽고 넘기지 말 것). 수정 후 같은 방식으로 재검증.
